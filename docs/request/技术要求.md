# 技术说明

## 1. 系统目标

你需要实现一个 **最小可运行的永续合约交易系统（Perpetual Futures Exchange）**。

系统需要包含以下模块：

- 前端（Frontend）
- 后端 API（Backend API）
- 交易引擎（Trade Engine）
- Vault 资金托管合约（Vault Smart Contract）
- 区块链事件监听器（Blockchain Indexer）
- 对冲系统（Hedging Bot）
- 数据库（Database）


### 系统架构示意

```
User
 │
 ▼
Frontend
 │
 ▼
Backend API
 │
 ├── Database
 │
 ├── Trade Engine
 │
 ├── Blockchain Indexer
 │
 └── Hedging Bot
          │
          ▼
    Hyperliquid

Vault Contract (onchain)
```

系统目标是实现一个 **完整的最小交易闭环**：

用户充值资金 → 用户开仓 → 系统自动对冲风险 → 用户平仓 → 用户提取资金。

技术架构可以自由设计。

# 2. 功能要求

## 2.1 钱包登录

用户需要通过加密钱包登录系统。

登录流程：

```
connect wallet
sign message
backend verify signature
create session/token
```

支持钱包：

- MetaMask（至少）
- 任意 EVM 兼容钱包（建议）

---

## 2.2 Vault 资金托管合约

实现一个 Vault 合约，用于托管用户资金。

需要支持的函数：

```
deposit
withdraw
```

资产类型：

```
USDC (ERC20)
```

合约必须发出事件：

```
Deposit(address user, uint256 amount)
Withdraw(address user, uint256 amount)
```

后端系统需要监听这些事件。

---

## 2.3 充值流程

用户充值流程：

```
approve USDC
deposit 到 vault
```

后端系统需要：

```
监听事件
更新用户余额
```

建议的余额模型：

```
available_balance
locked_balance
```

---

## 2.4 交易功能

交易所需要支持：

```
选择交易标的（symbol）
显示标的K线（price/kline）
设置杠杆（leverage）
设置保证金（margin）
开多 (open long)
开空 (open short)
平仓 (close position)
```

订单模型：

```
symbol
side
size
price
margin
leverage
```

要求：
symbol至少支持 **BTC**。
下单方式至少支持 **Market Order（市价单）**。

---

## 2.5 交易引擎

需要实现一个最小交易引擎。

必须的功能包括：

```
执行 market order
更新用户仓位
```
交易引擎可以是订单簿模式也可以是差价合约模式（CFD，参考极简合约）

标记价格的来源可以是：
- mock price（模拟价格）
- 外部 API
- oracle

执行价格根据交易引擎和对冲机制设计的不同，可以是不同的计算方式

---

## 2.6 仓位系统

需要维护用户仓位数据：

```
position_size
entry_price
unrealized_pnl
liquidate_price
```

系统需要支持：

```
加仓 (increase position)
减仓 (reduce position)
平仓 (close position)
清算 (liquidation)
```

---

## 2.7 对冲系统

交易所需要保持 **delta-neutral（风险中性）**。

当用户开仓时：

系统需要向 **Hyperliquid Testnet** 提交对冲订单。

示例：

```
用户做多 1 ETH，在CFD模式下（即用户与交易所资金池对赌），系统需要在 Hyperliquid上 做多 1 ETH，以转移风险敞口
```
当用户平仓时：

```
用户平多 1 ETH ，在CFD模式下，系统需要在 Hyperliquid上 平多 1 ETH，以匹配Net Position
```

对冲模块需要具备以下能力：

```
提交对冲订单
跟踪订单状态
处理失败情况（建议）
```
